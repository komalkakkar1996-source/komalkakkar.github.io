🧬 ONLINE SHOPPERS PURCHASING INTENTION ANALYSIS

Problem: E-commerce businesses struggle to understand why some visitors purchase while others leave without converting.

Action: Performed data cleaning and exploratory data analysis using Python, solved 18 business queries using SQL, and built an interactive Power BI dashboard to analyze customer behavior, traffic sources, bounce rates, and conversion trends.

Outcome: Identified that November and October had the highest conversion rates, returning visitors generated most conversions, and lower bounce rates strongly influenced purchases.

Recommendation: Improve landing page experience, focus marketing campaigns during peak conversion months, and invest more in high-performing traffic sources to increase conversions.

Tech Stack: Python, SQL, Power BI, Pandas, NumPy, Matplotlib, Seaborn

🧬 Online Shoppers Purchase Intention Prediction using Machine Learning

Problem: Predict which online shoppers are likely to complete a purchase despite class imbalance and diverse browsing behavior.

Action: Preprocessed the data, engineered features, built four machine learning experiments, and compared Logistic Regression, Decision Tree, and Random Forest models using hyperparameter tuning and evaluation metrics.

Outcome: The tuned Random Forest achieved the best performance with 89.3% Accuracy, 0.69 F1-Score, and 0.929 ROC-AUC, effectively identifying high-intent customers.

Recommendation: Recommended targeted campaigns for high-intent visitors, optimizing landing pages, and focusing promotions during peak conversion months to improve business revenue.

Tech Stack: Python • Pandas • NumPy • Scikit-learn • Matplotlib • Seaborn • Logistic Regression • Decision Tree • Random Forest

📊 Feature Selection Analysis for Student Exam Performance

Problem: Identifying the most important factors affecting student performance while eliminating redundant features to improve model efficiency and interpretability.

Action: Applied 9 feature selection techniques (Filter, Wrapper, and Embedded methods) and compared model performance before and after feature selection using Linear Regression.

Outcome: Reduced features from 23 to 17 while maintaining R² = 0.778, proving that fewer features can achieve similar predictive performance with better efficiency.

Recommendation: Recommended prioritizing high-impact features during data collection and use the optimized 17-feature model for future predictions and focus on key factors such as study hours, attendance, sleep quality, and study methods to improve student performance.

Tech Stack: Python • Pandas • NumPy • Scikit-learn • Matplotlib • Seaborn • Linear Regression • Feature Selection • RFE • RFECV • Lasso • Random Forest
